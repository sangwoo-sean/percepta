### 5.1 사용자 시나리오

#### 시나리오 A: 개인 블로거

| 항목            | 값   |
| --------------- | ---- |
| 월 피드백 세션  | 5회  |
| 세션당 페르소나 | 3명  |
| 요약 생성       | 매회 |

**월간 크레딧 사용량:**

- 피드백: 5 × 3 = 15 크레딧
- 요약: 5 × 1 = 5 크레딧
- **총 20 크레딧/월**

**월간 비용:** ₩200 (소량 패키지 기준) ~ ₩180 (기본 패키지)

## 11. 리스크 및 고려사항

### 11.1 API 가격 변동 리스크

- Google, OpenAI 등 가격 인상 가능성
- **대응**: 가격 변동 시 크레딧 단가 조정, 사전 공지

### 11.2 대량 사용자 원가 관리

- 비즈니스 패키지 할인율이 높아 마진 감소
- **대응**: 대량 사용자는 연간 계약 유도, 추가 가치 서비스 제공

### 11.3 무료 크레딧 남용

- 다중 계정으로 무료 크레딧만 사용
- **대응**: 이메일/전화 인증, IP 기반 제한

### 11.4 환율 변동

- API 비용은 USD로 결제
- **대응**: 분기별 가격 검토, 필요시 단가 조정

## 12. 결론 및 권장사항

### 12.1 핵심 인사이트

1. **AI API 원가 극히 저렴** - 피드백 1건당 ₩1 수준
2. **85~90% 마진 구조** - 사용량 기반에서도 높은 수익성
3. **낮은 진입 장벽** - ₩500부터 시작 가능
4. **확장성 우수** - 대량 사용자도 수용 가능

### 12.2 권장 크레딧 가격 체계
