# Percepta MVP 기능 정의

## 서비스 핵심 가치
> 고객의 시행착오를 획기적으로 줄이고, 제품 퀄리티를 높여주는 AI 페르소나 피드백 서비스

---

## Phase 1: 최소 기능 (1차 런칭)

### 1. 파일 업로드
- [ ] 텍스트 파일 (.txt)
- [ ] 마크다운 파일 (.md)
- [ ] 이미지 파일 (.png, .jpg) - 스크린샷 포함
- [ ] URL 입력 (랜딩페이지)

### 2. 페르소나 관리

#### 2-1. 페르소나 생성
- [ ] 조건 선택하여 N명 일괄 생성
  - 연령대 (20대/30대/40대/50대+)
  - 직업군 (개발자/마케터/디자이너/창업가/일반 직장인/학생)
- [ ] 생성 시 각 페르소나에 고유 이름/아바타 자동 부여
- [ ] 같은 조건이라도 성격/말투가 다른 개별 캐릭터로 생성

#### 2-2. 페르소나 목록
- [ ] 내 페르소나 전체 목록 보기
- [ ] 프로필별 통계 (예: 30대 마케터 3명, 20대 개발자 2명)
- [ ] 페르소나 삭제

### 3. 피드백 생성 및 조회

#### 3-1. 피드백 요청
- [ ] 업로드한 자료 선택
- [ ] 피드백 받을 페르소나 선택 (전체 또는 일부)
- [ ] 피드백 생성 실행

#### 3-2. 개별 피드백 (페르소나별 상세)
- [ ] 인터뷰 대화체 형식으로 출력
- [ ] 페르소나 프로필 (이름, 연령대, 직업) 표시
- [ ] 피드백 내용 예시:
  ```
  👤 김지현 (30대, 마케터)

  "오~ 이거 되게 깔끔하네. 랜딩페이지 첫 화면에서
  뭘 하는 서비스인지 바로 이해됐어.
  근데 가격 정보가 너무 아래에 있어서
  스크롤 안 했으면 못 봤을 뻔했네."
  ```
  ```
  👤 박준호 (20대, 개발자)

  "UI는 심플하고 좋은데, 가격 정책이 좀 애매하네.
  페르소나당 과금이면 많이 쓰면 비싸질 것 같은데...
  번들 할인 같은 거 있으면 좋겠다."
  ```

#### 3-3. 종합 피드백 (요약)
- [ ] 전체 페르소나 의견을 종합 분석
- [ ] 항목별 요약:
  - **긍정적 반응**: 공통적으로 좋게 평가한 점
  - **우려 사항**: 공통적으로 지적한 문제점
  - **개선 제안**: 구체적인 액션 아이템
  - **구매 의향**: 전체 N명 중 M명 긍정적

### 4. 인증 & 결제
- [ ] 소셜 로그인 (Google)
- [ ] 크레딧 충전 (Stripe 또는 토스페이먼츠)
- [ ] 페르소나 1명당 크레딧 차감 (피드백 생성 시)

---

## Phase 2: 확장 기능 (런칭 후 우선순위)

### 파일 지원 확장
- [ ] 엑셀 파일 (.xlsx)
- [ ] PDF 파일 (.pdf)
- [ ] 파워포인트 (.pptx)
- [ ] 스크린 녹화 영상 (.mp4, .webm)

### 페르소나 고도화
- [ ] 커스텀 페르소나 생성 (사용자 정의 특성)
- [ ] 세부 직업군 확장
- [ ] 국가/언어 확장
- [ ] 페르소나 성격 유형 선택 (긍정적/비판적/중립적)

### 피드백 고도화
- [ ] 피드백 히스토리 저장 및 버전 비교
- [ ] 후속 질문 기능 (특정 페르소나에게 추가 질문)
- [ ] PDF/이미지 내보내기
- [ ] 팀 공유 기능

## 화면 구성 (예상)

```
[대시보드]
├── 내 페르소나 (목록/생성)
├── 새 피드백 요청 (파일 업로드 → 페르소나 선택 → 생성)
├── 피드백 히스토리
└── 설정 (결제/계정)

[피드백 결과 화면]
├── 탭1: 종합 요약
└── 탭2: 개별 피드백 (페르소나별 카드 형태)
```

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React + Tailwind CSS |
| 백엔드 | NestJS |
| 데이터베이스 | PostgreSQL + TypeORM |
| 인증 | JWT (Passport.js) |
| 파일 저장 | AWS S3 또는 Supabase Storage |
| AI | Google Gemini / OpenAI GPT-4 / Claude API 중 선택 |
| 결제 | 토스페이먼츠 |
| 배포 | 프론트: Vercel / 백엔드: AWS 또는 Railway |

## 데이터 모델 (핵심)

```
User
├── id
├── email
├── credits

Persona
├── id
├── user_id (FK)
├── name (자동 생성)
├── avatar_url
├── age_group
├── occupation
├── personality_traits (JSON)

FeedbackSession
├── id
├── user_id (FK)
├── input_type (text/image/url)
├── input_content
├── created_at

FeedbackResult
├── id
├── session_id (FK)
├── persona_id (FK)
├── feedback_text (대화체)
├── sentiment (positive/negative/neutral)
├── purchase_intent (1-5)
```

## MVP 성공 지표

1. **기능 완성**: Phase 1 기능 100% 구현
2. **첫 결제**: 유료 사용자 1명 확보
3. **피드백 품질**: 사용자 만족도 4/5 이상

---

## 우선순위 원칙

1. **속도 우선**: 완벽함보다 빠른 출시
2. **핵심만**: 없으면 서비스가 안 되는 기능만
3. **검증 먼저**: 가설 검증 후 기능 확장
